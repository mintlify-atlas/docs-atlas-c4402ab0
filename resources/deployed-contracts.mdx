## Contract Architecture

StreamSwap uses a two-contract architecture:

1. **Controller Contract**: Manages the creation of new streams and handles protocol-level operations
2. **Stream Contract**: Implements the logic for managing individual token streams, including subscriptions, distributions, and finalization

Each controller address above can be used to create new stream instances. When you create a stream, a new stream contract is instantiated with your specified parameters.

## Using Contract Addresses

To interact with these contracts:

1. Choose the appropriate controller contract for your desired input token and network
2. Submit a `CreateStream` message with your stream parameters
3. The controller will instantiate a new stream contract and return its address
4. Users can then interact with the stream contract directly to subscribe, withdraw, and exit

For detailed integration instructions, see the [Integration Guide](/development/integration).
