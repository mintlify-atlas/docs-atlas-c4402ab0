# Quick Start

This guide will help you set up your development environment, build the StreamSwap contracts, and run tests.

## Prerequisites

Before you begin, ensure you have the following installed:

<Steps>
  <Step title="Install Rust">
    Install Rust using rustup if you haven't already:
    
    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```
    
    After installation, make sure your Rust installation is up to date:
    
    ```bash
    rustup update
    ```
  </Step>
  
  <Step title="Add WebAssembly Target">
    StreamSwap contracts compile to WebAssembly. Add the `wasm32-unknown-unknown` target:
    
    ```bash
    rustup target add wasm32-unknown-unknown
    ```
    
    <Info>
      This target is required to compile CosmWasm smart contracts.
    </Info>
  </Step>
  
  <Step title="Clone the Repository">
    Clone the StreamSwap contracts repository:
    
    ```bash
    git clone https://github.com/StreamSwapProtocol/streamswap-contracts.git
    cd streamswap-contracts
    ```
  </Step>
</Steps>

## Building the Contracts

<Steps>
  <Step title="Development Build">
    Navigate to the workspace directory and compile the contracts:
    
    ```bash
    cargo build
    ```
    
    This will compile all contracts and packages in the workspace:
    - `contracts/controller` - The controller contract
    - `contracts/stream` - The stream contract
    - `packages/types` - Shared type definitions
    - `packages/interface` - Contract interfaces
    - `packages/utils` - Utility functions
  </Step>
  
  <Step title="Run Tests">
    To ensure everything is working correctly, run the test suite:
    
    ```bash
    cargo test
    ```
    
    The test suite includes:
    - Unit tests for individual functions
    - Integration tests for contract interactions
    - Multi-contract workflow tests
    
    <Note>
      All tests should pass before proceeding to deployment.
    </Note>
  </Step>
  
  <Step title="Production Build (Optional)">
    For a production-ready optimized build, use the provided script:
    
    ```bash
    cargo run-script optimize
    ```
    
    This command:
    - Compiles contracts with optimizations enabled
    - Reduces binary size for on-chain deployment
    - Outputs optimized `.wasm` files to the `artifacts/` directory
    
    <Warning>
      The optimize script requires Docker to be installed and running.
    </Warning>
  </Step>
</Steps>

## Project Structure

After building, you'll find the following structure:

```
streamswap-contracts/
├── contracts/
│   ├── controller/          # Controller contract source
│   └── stream/              # Stream contract source
├── packages/
│   ├── interface/           # Contract interfaces
│   ├── types/               # Shared types and messages
│   └── utils/               # Utility functions
├── tests/                   # Integration tests
├── artifacts/               # Compiled .wasm files (after optimize)
└── Cargo.toml               # Workspace configuration
```

## Development Workflow

### Running Specific Tests

To run tests for a specific contract:

```bash
# Test controller contract
cargo test -p streamswap-controller

# Test stream contract
cargo test -p streamswap-stream

# Test with output
cargo test -- --nocapture
```

### Generating TypeScript Bindings

If you need TypeScript bindings for frontend integration:

```bash
scripts/schema.sh
```

This generates JSON schemas and TypeScript types for all contract messages.

### Code Generation

Generate contract schemas for queries and messages:

```bash
# Generate schema for controller contract
cargo run --bin controller-schema

# Generate schema for stream contract
cargo run --bin stream-schema
```

## Understanding the Contracts

<CardGroup cols={2}>
  <Card title="Controller Contract" icon="server">
    **Key Responsibilities:**
    - Creates new stream contracts
    - Manages protocol parameters
    - Collects creation fees
    - Handles governance operations
    
    **Main Entry Points:**
    - `instantiate` - Initialize the controller
    - `execute::CreateStream` - Create a new stream
    - `execute::UpdateParams` - Update protocol parameters
  </Card>
  
  <Card title="Stream Contract" icon="water">
    **Key Responsibilities:**
    - Manages individual token streams
    - Handles subscriptions and withdrawals
    - Performs continuous distribution
    - Executes finalization and post-stream actions
    
    **Main Entry Points:**
    - `execute::Subscribe` - Join a stream
    - `execute::Withdraw` - Withdraw unspent tokens
    - `execute::ExitStream` - Exit after stream ends
    - `execute::FinalizeStream` - Finalize stream
  </Card>
</CardGroup>

## Common Issues

<AccordionGroup>
  <Accordion title="cargo build fails with linker errors">
    Make sure you have the latest Rust toolchain:
    ```bash
    rustup update
    ```
    
    Also ensure the wasm32 target is installed:
    ```bash
    rustup target add wasm32-unknown-unknown
    ```
  </Accordion>
  
  <Accordion title="Tests fail with storage errors">
    Try cleaning the build cache:
    ```bash
    cargo clean
    cargo test
    ```
  </Accordion>
  
  <Accordion title="Optimize script fails">
    Ensure Docker is installed and running:
    ```bash
    docker --version
    docker ps
    ```
    
    The optimize script uses the `cosmwasm/workspace-optimizer` Docker image.
  </Accordion>
</AccordionGroup>
